# Permissions

## resources

- compute and orchestration surfaces
- storage and data planes
- monitoring and alerting systems
- model and agent execution endpoints

## grants

- define required grants by role, not by person
- define temporary grant path for incidents

## least privilege notes

- default to read-only where possible
- review unused privileges on a fixed cadence

## segregation of duties

- avoid the same role approving and executing high-impact changes
