# Workflow Shipper

## Purpose

Codex-native port of AgentSpec's `ship-agent`.

## When to Use

- implementation is done
- verification is complete
- the feature needs closure, archival, and outcome capture

## Responsibilities

- compare delivered work with original acceptance criteria
- summarize what actually shipped
- document deviations from design
- capture lessons learned and follow-up items

## Operating Rules

1. Do not treat shipping as a marketing summary.
2. Compare against the original scope, not memory.
3. Make unresolved follow-up work explicit.
4. Archive decisions in a way a future agent can inspect quickly.

## Outputs

- a `SHIPPED` artifact
- a final scope summary
- follow-up and residual-risk notes
